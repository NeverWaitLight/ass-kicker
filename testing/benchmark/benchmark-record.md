- 发送日志 MongoDB 3 次写入改为 1 次

### 优化 #3

**代码改进**：

- SendTaskExecutor 不再直接调用 SendRecordRepository，改为通过 SendRecordService.writeRecord 写入
- SendRecordService 新增 writeRecord(SendRecord)，内部维护缓冲池，满批（默认 100 条）或定时（默认 5 秒）触发批量插入 MongoDB
- 关闭时 DisposableBean.destroy 中 flush 剩余缓冲并 block 等待落库

### 优化 #4（已放弃）

**代码改进**：

- `SendRecordServiceImpl` 中将原有 buffer 缓冲池替换为 Disruptor 无锁队列

**压测结果**：

| 并发 | 实际TPS | P50(ms) | P95(ms) | P99(ms) |
| ---- | ------- | ------- | ------- | ------- |
| 1000 | 3010.7  | 291.3   | 457.9   | 2119.6  |
| 2000 | 2883.4  | 569.5   | 1153.8  | 3612.9  |
| 3000 | 2723.0  | 936.2   | 2372.2  | 5374.1  |
| 4000 | 2437.8  | 1196.2  | 3962.2  | 9165.2  |

**结论**：代码复杂度明显增加，但 TPS 与 P95/P99 相比优化 #3 的 buffer 方案略有下降或持平，收益不抵成本。**已放弃此优化**，维持原有 buffer 实现。
